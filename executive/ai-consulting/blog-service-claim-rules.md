# ブログ向けサービス範囲表現ルール

## 目的

ブログ記事内のCTAやサービス紹介で、AI活用診断簡易版・詳細版AI活用診断・導入支援・スポット相談の提供範囲を誤って約束しないようにする。

記事本文ではサービス内容を自由記述せず、原則として本書の定型CTAまたは同等の範囲に収まる表現を使う。

## サービス範囲の正本

| 区分 | 価格 | 含めてよい内容 | 含めてはいけない内容 |
|---|---:|---|---|
| AI活用診断簡易版 | 無料 | フォーム入力をもとにしたGoogle Slides簡易レポート。業種・規模に応じた活用方向性、AI活用が効果的な業務TOP3、ROI試算、コストレンジ、補助金名称 | MTG、無料コンサル、個別ヒアリング、設計レビュー、テスト実施、修正提案、脆弱性診断、修正実装、詳細な業務分析、アーキテクチャ図、導入支援見積 |
| 詳細版AI活用診断 | 5,500円税込 | MTGなしのGoogle Slides詳細レポート。詳細フォーム回答に基づく個別選定、アーキテクチャ図、具体額レンジ、ROI回収期間、導入支援の概算見積、導入可否・優先順位・費用前提 | MTG、面談、実装、テスト実施、脆弱性診断、修正実装、セットアップ伴走、導入作業、納品作業 |
| スポット相談 | 33,000円税込/枚 | 相談、要件整理、小規模作業、AI診断官レビュー | 無料提供、詳細版に同梱、導入支援費への自動充当 |
| 導入支援 | 個別見積 | 自動化システムの設計・構築・納品、API連携、初期動作確認、運用マニュアル | 定額の詳細版に含める表現、無料提供 |

## 禁止表現

### AI活用診断簡易版の近くで使わない

- 設計レビュー
- テスト実施
- 修正提案
- 脆弱性診断
- 修正実装
- 実装伴走
- セットアップ伴走
- テストゲート通過
- 具体的な構成案
- アーキテクチャ図
- 導入支援見積
- 個別ヒアリング
- 一緒に整理
- 無料MTG
- 無料コンサル

### 詳細版AI活用診断の近くで使わない

- MTG
- 面談
- ヒアリング
- 脆弱性診断
- テスト実施
- 修正実装
- 実装伴走
- セットアップ伴走
- 構築します
- 納品します
- 導入します

### 全記事で避ける

- 無料MTG
- 無料コンサル
- 詳細レポートに面談込み
- 詳細版に面談込み
- PDF納品
- PDFレポートで納品
- PDF/Excel/Word添付対応
- 詳細版で修正実装
- 詳細版で脆弱性診断

## 承認済みCTA

### 汎用

```md
AI活用をどこから始めるべきか迷っている場合は、まず [AI活用診断簡易版（無料）](/free-diagnosis) で、業種・規模に合った活用方向性と効果の目安をご確認ください。より具体的に整理したい場合は、[詳細版AI活用診断（¥5,500税込・MTGなし）](/free-diagnosis?paid=1) で、構成案、優先順位、費用前提を整理してお届けします。
```

### セキュリティ不安向け

```md
公開前チェックに不安がある場合は、まず [AI活用診断簡易版（無料）](/free-diagnosis) で、業務上の利用目的と最低限確認すべきセキュリティ論点を整理できます。

御社の状況に当てはめた優先順位、確認観点、概算の対応方針を知りたい場合は、[詳細版AI活用診断（¥5,500税込・MTGなし）](/free-diagnosis?paid=1) をご利用ください。ソースコードや実環境を対象にした検査・修正作業は、導入支援またはスポット相談として個別にお見積もりします。
```

### 導入支援向け

```md
具体的な業務自動化の構築まで進めたい場合は、まず [詳細版AI活用診断（¥5,500税込・MTGなし）](/free-diagnosis?paid=1) で、導入可否、優先順位、構成案、費用前提を整理します。実装・API連携・初期動作確認は、導入支援として個別にお見積もりします。
```

### スポット相談向け

```md
レポートの解釈や導入前の論点整理を人と確認したい場合は、詳細版AI活用診断とは別に、スポット相談またはAI診断官レビューをご利用いただけます。
```

## 運用ルール

1. 新規ブログ・既存ブログ修正時は、CTAを本書の承認済みCTAから選ぶ。
2. 承認済みCTA以外を書く場合は、`executive/ai-consulting/無料版vs有償版_定義.md` と `executive/ai-consulting/用語マスタ.md` に照合する。
3. 公開前に `npm run check:blog-service-claims` を実行する。
4. 警告が出た場合は、本文を修正するか、正本に基づいて意図的に許容する理由をファクトチェック記録へ残す。
5. 無料版・詳細版の範囲変更がある場合は、本書、用語マスタ、無料版vs有償版定義、チェックスクリプト、該当ページを同時に更新する。
