# 有償版 詳細レポート 運用フロー設計書 v1.0

> 2026-05-15追記: 本書は CEO 承認ボタンを前提にした旧設計を含む。現行方針は「人間チェックなしの完全自動納品」であり、最新の運用手順は `executive/ai-consulting/有償版_完全自動運用手順_v1.0.md` を正とする。詳細版の申込受付は公開済みで、旧申込停止表示は解除済み。

## 1. 目的

【詳細版】AI活用診断レポート（¥5,500 税込）を「銀行振込確認 → CEO 承認ボタン → AI 生成 → Google Slides 共有」のワークフローで運用する。

簡易版と同じく **Google Slides テンプレを 1 回作って、申込ごとにコピーして共有** する方式。PPTX は初回テンプレ作成時のみ使う。

## 4. 各フェーズの詳細

### Phase A：申込受付（既存改修）

- `/free-diagnosis` の paid タブは公開済み
  - `src/pages/api/free-diagnosis.ts` の申込停止ガードは解除済み
  - フォーム側の申込停止アナウンスは削除済み
- 申込後、自動で「振込先案内メール」を送信
  - 振込先：GMO あおぞらネット銀行 フリー支店（普通）1211110 / ゴウドウガイシャオプティエンス
  - 振込人名義は「申込番号 + 会社名」を依頼（消し込みの容易化）

### Phase B：入金確認（CEO の手動運用）

- freee で入金検知（既に freee MCP 連携済み）
- もしくは GMO あおぞらネット銀行で目視確認
- 入金が確認できたら `/admin/leads/[id]` で「レポート生成」ボタンを押す
  - ボタン押下で leads.status を `'paid_verified'` に変更
  - これが Edge Function 起動のトリガーになる

### Phase C：レポート自動生成（Edge Function：要新規実装）

新規 Edge Function：**`process-paid-diagnosis`**

#### 入力
- `leads.id`（DB webhook 経由）
- `leads.status === 'paid_verified'` のみ処理（誤発火防止）

#### 処理ステップ
1. 月次上限チェック（簡易版とは別カウント、上限は 10 件/月程度に設定）
2. OpenAI で詳細版レポート内容を生成（JSON 構造）
   - スキーマ：簡易版より大規模（19 スライド分のプレースホルダーを埋める）
   - means / staff_level タグも継承
3. JSON Schema バリデーション
4. Google Slides API でテンプレ（`1DzM-D5sncQNFpvre0b785NdJK2MF4u4fta8o27SLe54`）をコピー
5. プレースホルダー置換（19 スライド分）
6. 共有設定（anyone with link / viewer）
7. Resend で完成メール送信（顧客 + 領収書 PDF or 別途送付）
8. `leads.status = 'completed'`, `slides_url`, `sent_at` を更新

#### スキーマ設計（簡易版を拡張）
詳細版 PPTX で使う `{{paid_*}}` プレースホルダー全 90 個程度に対応する JSON 構造を `executive/ai-consulting/有償版_詳細レポート設計書_v1.0.md` に基づいて定義。

#### 環境変数（追加）
```
GOOGLE_SLIDES_PAID_TEMPLATE_ID=1DzM-D5sncQNFpvre0b785NdJK2MF4u4fta8o27SLe54
```

### Phase D：納品後フォロー

- レポート完成メール内に「必要に応じてAI診断官レビューをご案内できます」と記載する
- AI診断官レビューは有償版レポートとは別導線で扱い、固定の人間時間を有償版に含めない

## 5. 管理画面の追加要件

`/admin/leads/[id]`（既存）に以下を追加：

| ボタン / 表示 | 条件 | アクション |
|---|---|---|
| 「レポート生成」ボタン | `status='payment_pending'` | クリックで `status='paid_verified'` に変更 → webhook 経由で Edge Function 起動 |
| 入金確認チェックボックス | 任意 | `payment_confirmed_at` を記録（監査用） |
| 振込通知欄 | 表示のみ | 振込人名義・金額・日時の手動メモ欄 |

## 6. ステータス遷移

```
draft         （フォーム送信直後）
  ↓
verified      （簡易版のメール認証後 / 詳細版では使わない）
  ↓
payment_pending    （詳細版：申込済・入金待ち）
  ↓
paid_verified      （詳細版：CEO がレポート生成ボタンを押した状態）
  ↓
processing    （Edge Function 実行中）
  ↓
completed     （レポート送付完了）

エラー時：
  ↓
manual_review     （バリデーション失敗等）
quota_retry_pending  （OpenAI クォータ超過）
```

簡易版は `verified → processing → completed`、詳細版は `payment_pending → paid_verified → processing → completed`。

## 7. 実装の優先順位（MVP→ 完成形）

### MVP（最初の 1-2 件運用）
1. `/api/free-diagnosis` は詳細版申込を受付済み。現行ステータスは `pending_payment` を使う
2. 振込先案内メールテンプレ作成
3. `/admin/leads/[id]` に「レポート生成」ボタン手動追加
4. Edge Function `process-paid-diagnosis` 新規作成（簡易版をベースに）
5. 詳細版用 JSON Schema 定義

### 拡張（運用が定着してから）
- freee API で入金自動検知 → CEO に Slack/メール通知
- 領収書 PDF 自動生成（適格請求書）
- 月次上限とリトライ機構（簡易版と同等）
