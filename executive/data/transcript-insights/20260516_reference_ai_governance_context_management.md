# 社内AI制限下でのコンテキスト管理とシャドーAI抑制

## 出典

- 2026-05-16 CEO共有の動画文字起こし
- テーマ: 会社で使えるAIが Microsoft 365 Copilot や Gemini for Google Workspace に限定される場合のAI活用、コンテキスト収集、社内調整、シャドーAIリスク

## 中核メッセージ

AI活用の品質は、モデル選定だけでなく「業務コンテキストをどれだけ継続的に集め、許可された環境で扱えるか」に大きく左右される。社員が個人AIや未承認ツールへ流れる背景には、社内で使えるAI環境の不足、ルールの曖昧さ、議事録・資料・判断履歴の欠落がある。Optiensの記事化では、抜け道の推奨ではなく、シャドーAIを生まない社内ルールと運用導線として再構成する。

## Optiens への適用ポイント

- AI導入支援では、ツール導入より先に「どの情報をAIに渡してよいか」を分類するルール作りを提案する。
- Microsoft 365 Copilot / Gemini for Google Workspace など、顧客がすでに契約している環境を前提にした小さな導入ステップを用意する。
- 議事録、提案書、問い合わせ履歴、社内FAQ、決裁メモをコンテキスト資産として扱い、保存場所・権限・保持期間を先に決める。
- 「AIを禁止する」だけではシャドーAIが増える可能性があるため、許可されたツール、禁止データ、相談窓口、例外申請をセットで設計する。
- 経営者向けには費用対効果、情報システム向けには権限・監査・DLP・データ処理条件を分けて説明する。

## 採用しない要素

- 個人端末・個人AIへ業務情報を持ち出すように見える示唆は採用しない。
- 会社規程の「グレーゾーン」を攻める表現はOptiensのブランドトーンに合わないため不採用。
- 特定YouTubeチャンネル、出演者、口癖、宣伝CTA、コメント紹介部分は記事本文に残さない。
- 「Copilot/Geminiなら何でも入れてよい」という断定はしない。契約、管理者設定、社内規程、データ分類によって判断が変わるため条件付きで扱う。
