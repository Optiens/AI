# AI活用支援 — 業務フロー定義

## v1.6 更新内容（2026-05-15）

### AI活用レビュー面談と有償要件定義の差別化

- AI活用レビュー面談は、無料診断または詳細版AI活用診断後の相談サービスとして扱う。レポート解説、優先業務の絞り込み、対応可否、概算費用帯、着手可能時期、次アクションを整理する
- AI活用レビュー面談は導入支援費へ充当しない。導入支援や有償要件定義へ進む場合は、面談で得た情報を引き継ぎ、重複ヒアリングを減らし、次工程の提案精度を上げる
- 有償要件定義は、導入支援前の有料スコーピングとして扱う。キックオフMTGではない
- 有償要件定義は、対象業務、実装範囲、前提条件、含む範囲・含まない範囲、概算費用、次アクションを整理する
- 導入支援に進んだ場合、初期費用へ充当するのは有償要件定義のスポットチケット1枚分のみ。AI活用レビュー面談は別会計とする
- サービスメニュー上は「相談・要件整理」枠に、AI活用レビュー面談と有償要件定義を並べて含める。スポットチケットはサービス名ではなく支払い単位として補足する

全UIから削除し、「業務分析・レポート作成」「業務分析・提案作成」へ統一。
理由: AI単独実施に見え、コンサルタントの介在価値が低く見積もられる

### 案件別の概算見積を有償レポートに導入（v1.4新設）
有償レポートの自動化提案（5〜8件）に対し、**案件単位で概算見積** を併記する。
- バッファ方針: **素工数 × 1.7〜2.0倍**（Phase A1。実績10件後に1.5倍へ引下げ可）
- 提示形式: 「3パターン提示」（シンプル版・標準版・フル版）を可能なら採用
- 前提条件・含む範囲・含まない範囲・リスク要因を **必ず併記**
- 「正式見積は導入支援契約前に無料MTGで確定」と明記

### 地元小規模事業者支援の方針（価格は下げず、入口を下げる）
価格割引は持続不可能。代わりに以下3つで導入ハードルを下げる：
1. **段階導入推奨**: Phase 1パイロット（15〜30万円） → 効果実感 → 拡張
2. **テンプレ化サービス**: よくある業務パターン（問合せ振分・見積書生成等）を1業務15〜30万円固定
3. **補助金活用案内**: レポートに名称のみ記載（申請代行は業務範囲外）

「地元なので割引」は禁止。一度割引すると次も期待され持続不能。

# AI活用支援 — 業務フロー定義

## 目的
顧客接点ごとの対応手順を定義し、属人化を防ぎ、再現可能な営業・納品オペレーションを構築する。
本書は会社資産として運用し、改訂時はバージョンを上げる。

v1.2では、4エージェント並列レビュー（review-agent / strategy-agent / business-analyst / critical-analyst）の指摘39件と、CEO決定（プラン体系再編・出力フォーマットGoogle Slides化・CRM=Supabase Edge Functions＋pg_cron・「相殺」→「充当」）を統合し、「ハッピーパス」一辺倒から「異常系・段階的運用条件」を併載する構造に転換した。

## 戦略目標

- **Pattern B/C比率50%以上を維持する**。Pattern Aは入口・教育目的、収益柱はB/C。
- Pattern A単体では時給換算で月50万報酬を支えきれない（business-analyst試算）。Aの存在意義は「リード品質フィルター・教育・実績アピール材料」。
- Phase A1（〜6ヶ月）目標: 有償案件1件
- Phase A2（7-12ヶ月目）目標: 月1-2件受注、月間売上15-30万円
- Phase A3（2年目）目標: 月3-5件、月間売上50-100万円
- Phase A4（3年目〜）目標: 月間利益40-100万円

### 重要KPI閾値

| KPI | 推奨閾値 | 用途 |
|---|---|---|
| 無料→¥5,000転換率 | 15%以上（3ヶ月平均） | フォーム改善・無料版差別化の見直し閾値 |
| ¥5,000→導入支援転換率 | 20%以上 | レポート品質・MTGクロージングの見直し閾値 |
| スポットチケット1枚要件定義不成約率 | 50%以下 | 提案精度の見直し閾値 |
| 検収後3ヶ月無償サポート | 累計10h上限 | 超過時は保守契約必須化 |
| 保守ライト超過率 | 20%以下 | 超過頻発時はスタンダード推奨 |
| 導入支援最低受注額 | 40万円 | これ未満は¥5,000レポートに誘導 |
| 同時案件並走上限 | 暫定3件 | キャパ崩壊防止（Phase A1実績で改定） |
| スポット相談月受付上限 | 5件/月 | キャパ保護 |
| 応答SLA | 3営業日以内 | ワンオペ前提・常時オン否定 |
| 賠償上限 | 契約金額の1倍 | 業界相場下限・契約書明記 |

## Definition of Ready（DoR）— 全Pattern稼働の前提

DoR（Definition of Ready）= 「v1.2を本番運用扱いに格上げするための必須前提」。
DoR充足前は **Phase A1暫定運用**（口頭合意ベース）として案件を進めることをCEO判断で許容するが、契約書テンプレート完成を最優先する。

### DoRチェックリスト

- [ ] 要件定義業務委託契約書テンプレート完成（弁護士確認済）
- [ ] AI導入支援業務委託契約書テンプレート完成（弁護士確認済）
- [ ] 紹介者契約書テンプレート完成（Pattern C用）
- [ ] 特商法表記サイト掲載済（B2B限定の旨）
- [ ] プライバシーポリシー掲載済
- [ ] 充当時の消費税処理を税理士確認済
- [ ] 不成約時の成果物（要件定義レポート）の著作権帰属確定
- [ ] 全成果物に「Optiens独占的設計に基づく・第三者への開示転用禁止」明記
- [ ] 全自動メールに配信解除リンク付与（特定電子メール法対応）
- [ ] フォームDoS対策（reCAPTCHA・レート制限・手動承認段階）実装済

### DoR未充足時の暫定運用

- 全Pattern暫定運用OK（CEO決定）。ただし「Phase A1暫定運用」と業務メモに明記し、契約書テンプレート完成を最優先タスクとする。
- 暫定運用中の案件は、契約書完成後に**遡って正式契約書で再締結する**ことを顧客と合意しておく。

## Pre-Funnel判定ルール（流入経路別の自動振り分け）

cold MTGリクエスト・直接問い合わせ・各種フォーム入力に対し、自動でA/B/C/C-softに振り分ける。
**cold MTGリクエストはfunnel強制誘導**（自動返信メールでA/B/Cプラン提示）。

### 判定フロー

```
[流入]
 │
 ├─ サイト「無料AI診断」フォーム送信
 │    → Pattern A（標準フロー・無料診断スタート）
 │
 ├─ サイト「詳細レポート申込」フォーム送信
 │    → Pattern A の途中（ステップ4以降）に合流
 │
 ├─ MTG予約フォームから直接予約（cold）
 │    → 自動返信メールでA/B/Cプラン提示し誘導（funnel強制誘導）
 │    → 予約は受けない（Pattern Aの無料診断 or Pattern Bのスポットチケット1枚の要件定義 or Pattern Cの紹介経由を選んでもらう）
 │
 ├─ メール・電話で「直接導入支援を頼みたい」
 │    → Pattern B（スポットチケット1枚の有償要件定義経由）
 │
 ├─ 商工会・既存顧客・営業パートナー経由の紹介
 │    │
 │    ├─ 紹介者がOptiensを名指し → Pattern C 標準（無料キックオフMTG継続）
 │    └─ 紹介者が「AIに詳しい会社を紹介」レベル（温度感低い） → Pattern C-soft → Pattern A合流
 │
 └─ 完了後の再相談
      └─ スポットチケット / 保守チケット / 個別見積のいずれかで対応
```

### freee API ファジーマッチの注意点

- 振込人名と申込時会社名が **大きく異なる場合** はマッチしない（例: 個人名で振込）
- マッチしない振込は管理画面の「pending_payment」リストに残る
- CEO は管理画面で `/admin/leads/<id>` の **「お振込確認済みに変更」** ボタンで手動承認可能
- 重複マッチ防止: 同じfreee取引IDは1度しか使われない（usedTxnIds で管理）

```

### 重要: 顧客に届くメールは2通になる
- 1通目: 振込案内メール（自動・申込時）

- 2通目に「先にお送りした振込案内は取り消しとなります」を明記して混乱を防ぐ

- Phase A1（紹介数 年数件想定）では事前発行URL方式の管理コストに見合わない
- テスト用途・補償・goodwill 対応にも同じボタンで対応可能
- 紹介者は「Optiensに直接お申込みください」と顧客に伝えるだけでOK

## 紹介経由・無料化の運用ルール

### 適用条件

| ケース | 無料化 | 用途 |
|---|---|---|
| Pattern C 標準（紹介者がOptiensを名指し） | **無料化** | 紹介者経由顧客の優遇・紹介者への謝意 |
| Pattern C-soft（温度感低い紹介） | 無料化しない | Pattern A無料版に合流 |
| 商工会・自治体イベント参加者 | ケースバイケース | CEO判断 |
| 既存顧客のリピート紹介 | **無料化** | 関係維持 |

### CEO作業手順

1. 顧客が `/free-diagnosis` 有償版で申込 → CEO に通知メール届く
2. 紹介リストと照合（紹介経由か判断）
3. 紹介経由の場合：`/admin/leads/<id>` 詳細画面の **「紹介経由・無料化」** ボタンをクリック
   - モーダルで紹介者名（任意）入力 → 実行
   - 自動で status=paid + 顧客に「お振込不要」メール送信
4. 紹介経由でない場合：何もせず freee API 自動検知を待つ

### 管理画面の使い方

- パスワード認証: `ADMIN_PASSWORD` 環境変数
- URL: `/admin/login` → ログイン後 `/admin/leads`
- パイプライン表示・テーブル表示の切替可
- 各リード/行クリックで詳細・編集画面 `/admin/leads/<id>`
- 編集画面で全フィールド編集可、ステータス変更可、各種アクション実行可

### 全体フロー（先払い型）

```

## Pattern B: スポットチケット1枚の有償要件定義フロー（v1.3で更新）

### 適用条件
- 顧客が「詳細レポートは不要、導入支援を直接スタートしたい」と明示した
- すでにAI活用の方向性を自分で固めている層

### v1.2変更点（CEO決定）

- 旧「無料ヒアリング（60-90分）」フローを **廃止**
- 「直接導入支援希望」の顧客は **スポットチケット1枚（¥33,000税込、税抜¥30,000）の有償要件定義経由** で受ける
- 旧「1顧客1件・3ヶ月クールダウン」判定ルールも廃止（スポットチケット1枚の有償フィルタが代替）
- escape hatch（緊急時別ルート）は残す（CEO単独判断で例外的に無料相談を受ける場合は議事録に「escape hatch適用」と明記）

### 根拠
- スポットチケット1枚の本質は「直接導入支援希望者向けの本気度フィルター + 要件定義工数の対価」。法的には準委任契約として、業務完了時に報酬請求権が確定する（不成約時も返金不要）
- 成約時は本契約の初期費用に充当（旧表記「成約時相殺」を「初期費用に充当」に統一・ip-agent法務調査結果反映）

ip-agent法務調査§2-2準拠。契約書には必ず以下の条項を含める:

- 同一顧客の再依頼で要件定義が完全に作り直しになる場合は、再度スポットチケット1枚分を請求
