# 格安データベースとSQLite系クラウドDBの選定

## 出典

- CEO共有のプログラミング系対談文字起こし（2026-05-16共有）
- 公開記事では、元動画名、チャンネル名、話者名、ラジオネーム、宣伝要素、固有の雑談、発音談義を使用しない。

## 中核メッセージ

AIコーディングで小さなアプリを大量に試作できるようになるほど、1アプリごとの固定インフラ費が重くなる。Cloudflare D1やTursoのようなSQLite系クラウドDBは、低コストPoCや軽量業務ツールの選択肢になる一方、書き込み性能、SQL機能差、マイグレーション、テナント分離方式の設計を事前に見る必要がある。

## Optiensへの適用ポイント

- AI支援事業の軽量PoCでは、Supabase/PostgresだけでなくD1やTursoを候補に入れる。
- 顧客の本番業務データ、複雑な権限、分析クエリ、監査要件がある場合は、Postgres系を標準候補にする。
- 低コストのデモ、単機能の社内ツール、読み取り中心の小規模アプリでは、D1やTursoの採用余地がある。
- 「1顧客1DB」の分離は、ロウレベルセキュリティの複雑さを下げる可能性があるが、全DBへのマイグレーション運用が難しくなる。
- 公式価格・制限は変わりやすいため、提案書では「2026年5月時点」「公式料金ページ確認」を明記する。
- 有償レポートでは、DB選定を「価格」だけでなく、書き込み頻度、データ量、権限、将来の移行性で評価する。

## 採用しない要素

- 元動画の料金断定や発音談義はそのまま使わない。
- 「D1/Tursoが無敵」「Supabaseは高すぎる」といった煽りは採用しない。
- SQLiteは本番で常に十分、という断定はしない。
- ユーザーごとにDBを分ける方式を万能策として扱わない。
