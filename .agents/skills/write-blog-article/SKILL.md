# ブログ記事自動生成スキル

このスキルはOptiens公式ブログの記事を自動生成し、デプロイまで行います。

## 実行手順

### Step 1: テーマ選定
以下の7カテゴリからローテーションでテーマを選定する。前回の記事カテゴリと重複しない。
既存記事で最も少ないカテゴリを優先的に選ぶ（カテゴリバランスを維持）。

| カテゴリ | 目的 | テーマ例 | ターゲット読者 |
|---|---|---|---|
| AI | **メイン事業の集客の入口**。地方の小規模事業者がAIを使うメリット・始め方を伝える | AI活用診断の事例紹介、業種別AI活用Tips、コスト削減の数値例、無料で試せるAI活用法 | AI導入を検討する小規模事業者・個人事業主 |
| business | 小規模事業者の経営・DX全般 | 補助金活用、業務効率化、freee活用、LINE公式活用、集客戦略 | 経営者・個人事業主 |
| technology | IoT×水耕栽培+AI技術の技術力をアピール | IoTセンサー構築、Raspberry Pi活用、Zigbee/MQTT、LED制御、MCP | 技術導入を検討する事業者・エンジニア |
| agriculture | 水耕栽培の魅力・始め方（長期事業の種まき） | 品目別栽培ガイド、収益シミュレーション、植物工場トレンド | 水耕栽培に興味がある人・検討者 |
| welfare | 農福連携・就労支援での水耕栽培活用 | 就労支援事業所での導入メリット、作業設計、工賃向上 | 就労支援事業所・自治体（Phase H5の直接ターゲット） |
| social | 社会課題×テクノロジー | フードマイレージ、遊休施設活用、地産地消、食料安全保障 | 社会課題に関心がある層 |
| announcement | 事業進捗 | マイルストーン達成、新サービス、パートナーシップ | 既存フォロワー・見込み顧客 |

### カテゴリ戦略（AI支援事業を軸にしたコンテンツマーケティング）
ブログの主目的は「AI活用に興味がある小規模事業者を集客→無料診断→有償案件の受注」。
水耕栽培は長期事業として並行発信し、将来のPhase H5（導入支援）への布石とする。

- **AI/business**: メイン。「AIで業務を楽にしたい事業者」を集める記事群（全体の50%目安）
- **technology**: Optiensの技術力を示す記事群（AI支援・水耕栽培の両方をカバー）
- **agriculture/welfare**: 水耕栽培事業の長期コンテンツ。Phase H5に向けた種まき
- **social**: 社会的意義を示す記事。ブランディング
- **announcement**: 事業の実在感・進捗

### ローテーションルール
投稿頻度は柔軟に（週1-3回）。以下の比率を目安にカテゴリバランスを調整:
- **AI/business**: 全体の50%（最優先）
- **technology**: 全体の20%
- **agriculture/welfare/social**: 全体の25%
- **announcement**: 必要時のみ（5%）

テーマ選定時は以下のエージェントを使用:
1. **researcher**エージェントで最新のAI活用・業務効率化ニュースを調査
2. 既存記事（`src/content/blog/`）と重複しないテーマを選定
3. SEOキーワードを意識（「AI 業務効率化 中小企業」「AI活用 飲食店」「水耕栽培 IoT」等）

### Step 2: 記事執筆
以下の構成で2000-3000字の記事を作成:

```
---
title: '記事タイトル（SEOを意識、30-60字）'
date: 'YYYY-MM-DD'（当日の日付）
category: 'AI' | 'agriculture' | 'business' | 'technology' | 'welfare' | 'social' | 'announcement'
excerpt: '要約（100-150字）'
image: '/images/blog/スラッグ名.webp'
---

## 導入（なぜこのテーマが重要か）
## 本論（2-3セクション、データ・事例を含む）
## Optiensの取り組み（自社の関連性を自然に言及）
## まとめ
```

### Step 3: ファクトチェック（最重要）

**3a. プロジェクト進捗との整合性チェック**
1. `src/content/blog/_project-status.md` を必ず読み、現在のフェーズ・完了済み項目・未完了項目を把握する
2. 記事内のOptiensに関する記述が進捗状況と矛盾していないか検証する
3. 未完了の項目を完了済みかのように記述していないか確認する
4. 禁止表現リストに該当する表現がないか確認する

**3b. その他のファクトチェック**
- **統計データ**: 出典URLが有効か確認（WebFetchで検証）
- **技術的記述**: 正確性を確認（スペック値、プロトコル名等）
- **禁止事項**: AGENTS.mdの禁止表現が含まれていないか確認
- **日付・数値**: 矛盾がないか確認

### Step 4: アイキャッチ画像生成（必須・スキップ禁止）

**正規スクリプト**: `scripts/generate-blog-imagen.mjs`（Imagen 4 Ultra / Google Vertex AI）

```bash
node scripts/generate-blog-imagen.mjs "記事テーマに合わせた英語プロンプト" "public/images/blog/<記事ファイル名と同じスラッグ>.webp"
```

**重要・パス命名ルール**（過去事故の再発防止）:
- 画像ファイル名は **記事 md のファイル名（拡張子除く）と完全一致** させる
- 例: `src/content/blog/20260508-foo-bar.md` → `public/images/blog/20260508-foo-bar.webp`
- frontmatter `image:` も同じパスを書く: `image: '/images/blog/20260508-foo-bar.webp'`
- 日付プレフィックスを **片方だけ落とす運用は厳禁**（参照不一致で本番表示が壊れる）

仕様:
- モデル: `imagen-4.0-ultra-generate-001`
- アスペクト比: 16:9
- PNG → WebP 変換（sharp 使用）
- `GCP_PROJECT_ID` と `GOOGLE_APPLICATION_CREDENTIALS` を `.env` から自動読込

フォールバック:
- Vertex AI / Google Cloud クレジット / 認証で失敗した場合のみ `scripts/generate-blog-gemini.mjs`（Gemini 2.5 Flash Image）を検討する
- `scripts/generate-blog-openai.mjs` は旧gpt-image-2運用のレガシー。明示指示がない限りブログアイキャッチの正規手順には使わない

画像プロンプトのルール:
- 英語で記述
- テキスト・文字・ロゴ・数字を含めない（`No text, no letters, no numbers`）
- モダン・クリーン・プロフェッショナル（editorial business magazine）
- ブランドカラー指定はしない（記事テーマに応じた自然な配色をAIに任せる）
- AI支援系: 業務シーン・デジタル化・抽象的な情報フローのビジュアル
- 水耕栽培系: 自然×テクノロジー・IoT・ハーブのビジュアル

**必須運用ルール**:
- 記事 md を作成したら **必ず同セッション内で画像も生成する**
- 画像生成を後回しにしない（後でまとめてやろうとして忘れる事故の防止）
- 生成失敗時は CEO に報告。スキップは可能だが必ず明示する

### Step 5: ファイル保存
- 記事: `src/content/blog/{slug}.md`
- 画像: `public/images/blog/{slug}.webp`

### Step 6: ビルド確認
```bash
npx astro build
```
エラーがないことを確認。

### Step 7: Git commit & push
```bash
git add src/content/blog/{slug}.md public/images/blog/{slug}.webp
git commit -m "blog: {記事タイトルの要約}"
git push origin main
```

## 注意事項
- AGENTS.mdの禁止事項を必ず遵守
- 他サイトの文章をコピーしない（参考にしてオリジナルで書く）
- 出典は記事末尾にまとめる
- 画像生成に失敗した場合は画像なしで記事を公開（imageフィールドを省略）

## 【必須】事実確認ルール（フェーズ整合性）

### AI支援事業の現状（Phase A1）
- 無料AI診断のLP・フォーム・サンプルレポート9業種は公開済み
- 有償案件の受注実績はまだない
- 「初受注を目指している段階」が正確な表現

### 水耕栽培事業の現状（Phase H1 — 開始前段階）
以下のルールを必ず守ってください。

### 禁止表現（事実に反する）
- 「センサーデータを収集しています」「モニタリングしています」など、システムが現在稼働中であるかのような現在形
- 「〇〇件のデータが蓄積されています」など、具体的な実績数値を記載すること（架空データ禁止）
- 「バジルを収穫しました」「〇〇gの収穫ができました」など、栽培・収穫の実績を記載すること
- 「レストランに納品しています」など、販売実績を記載すること

### 許可される表現（設計・計画中）
- 「〇〇する構成を設計しています」「〇〇する予定です」「構築中です」
- 「〇〇を目標にしています」「〇〇を想定しています」
- 技術解説記事として「こうすれば実現できる」という一般論・実装方法の解説
- AI支援事業: 「9業種のサンプルレポートを公開しています」「無料診断を提供しています」はOK

### 判断基準
「Optiensでの取り組み」「Optiensでの実運用」などの自社紹介セクションで
現在進行形を使う場合は、事業ごとに正確な段階を記述する。
実績データ（収集件数・収穫量・売上）は一切記載しない。
