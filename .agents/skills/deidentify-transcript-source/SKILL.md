# 文字起こし元ソース匿名化・再構成スキル

文字起こしは「素材」であり、記事の骨格としてそのまま使わない。読者が元動画・元投稿・話者を推測できる状態で公開してはいけない。

## 必須ルール

- 動画タイトル、チャンネル名、話者名、イベント名、コミュニティ名、宣伝CTA、口癖、決め台詞を本文に残さない。
- 元動画の項目順、見出し順、デモ順、例示順をそのままなぞらない。
- 文字起こしの長い言い回し、比喩、煽り、固有のフレーズを使わない。
- 元動画だけに出てくる具体例をそのまま使わない。Optiensの顧客文脈に置き換える。
- 公式発表・一次情報で確認できる内容だけを事実として扱い、動画は出典として本文に書かない。
- 記事の主語は「動画の内容」ではなく、Optiensの読者が抱える業務課題に置き換える。

## 再構成手順

1. **素材抽出**
   - 文字起こしから、事実、示唆、論点、使える例、使わない宣伝要素に分ける。
2. **一次情報化**
   - 固有名詞、機能名、日付、提供条件、数値は公式情報で確認する。
   - 公式確認できない動画由来の主張は削除または「可能性」表現に弱める。
3. **構成変更**
   - 元動画の順序と違う記事構成にする。
   - 例: `ニュース紹介 → デモ → 4原則 → 感想` を、`中小企業の業務課題 → 何が変わるか → 導入時の注意点 → Optiens視点の使いどころ` に変える。
4. **例の置換**
   - 元動画のデモ例や固有例を、問い合わせ対応、見積作成、社内検索、議事録、Google Workspace、CRMなどOptiens読者向けの例に置き換える。
5. **Optiens独自視点の追加**
   - 中小企業での使いどころ、導入リスク、運用ルール、費用対効果、セキュリティ、業務定着の観点を1つ以上追加する。
6. **検索耐性チェック**
   - 7語以上連続して元文字起こしと一致する文を残さない。
   - 読者が本文の特徴的な一文を検索して元動画にたどり着けそうなら書き換える。

## 公開前チェックリスト

- [ ] 動画タイトル、チャンネル名、話者名、イベント名、宣伝CTAが本文にない
- [ ] 元動画の見出し順・デモ順・例示順をなぞっていない
- [ ] 文字起こし由来の特徴的な言い回しが残っていない
- [ ] 元動画固有の例をそのまま使っていない
- [ ] 公式一次情報で確認した事実だけを事実として書いている
- [ ] Optiens読者向けの独自視点・補足が入っている
- [ ] ファクトチェック記録に「元ソース匿名化チェック済み」と記録した

## 見送り基準

次の場合はブログ化せず、ナレッジ記録とCEO報告に留める。

- 文字起こしの内容が固有体験・宣伝・個人の主観に依存しすぎて、再構成すると価値が薄くなる
- 公式一次情報で中核事実を確認できない
- 既存記事と重複が大きく、独自視点を追加できない
- 元ソースを伏せるための改変が大きすぎて、実質的に別記事として成立しない
