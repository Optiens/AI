# GMOあおぞらネット銀行 法人API 調査（入金即時検知用途）

## 結論サマリ
- **無料**: 法人口座契約者は「スタンダードAPI（28種・無償）」で入出金明細照会・Webhook（イベント通知）を利用可能。月額・トランザクション課金なし。
- **即時性**: バーチャル口座 + Webhook で **24/365 ほぼリアルタイム入金通知**（数秒〜十数秒オーダー）。失敗時は最大1時間の指数バックオフ再送あり。
- **申請の手間**: オンライン申込フォーム → 事前ヒアリング → 接続審査 → 契約 → 開発環境テスト → 本番。**契約完了に最短1週間、本番接続まで約2ヶ月**が目安。本契約前に**サンドボックス「sunabar」を審査なし・無料**で即試せる。
- **推奨アクション**: 即 sunabar に登録してWebhookとOAuth2.0フローを検証 → 並行して本番API接続申込フォーム提出。Phase A2の入金照合自動化（freee・Supabase連動）に組み込む前提で進めれば追加コストゼロ。

## 1. 料金（無料か）
| 区分 | 内容 | 料金 |
|---|---|---|
| スタンダードAPI（28種） | 口座一覧・残高照会・**入出金明細照会**・振込入金明細・**Webhook（イベント通知）** | **無償** |
| オプションAPI（7種） | 高度機能（為替・与信系等） | 有償（個別） |
| バーチャル口座（振込入金口座） | リアルタイム入金検知に必須の入金専用口座 | 初期費用・月額**無料** |
| サンドボックス sunabar | 個人/法人問わず審査なしで利用可 | 無料 |

→ Optiens用途（入金検知）は **完全無料**で構成可能。

## 2. 即時性（レイテンシ）
- 「リアルタイム振込・入金サービス」は **24時間365日** 稼働、入金後ただちに残高反映・通知。
- 入金検知は **Webhook（イベント通知）方式** が公式提供。送信失敗時は **指数バックオフで最大1時間再送**。
- HMAC-SHA256でリクエストボディ署名 + アクセストークン検証 → 受信側Webhookエンドポイントで真正性確認可能。
- 注意：送金元金融機関側の処理タイミングにより、他行振込は着金が翌営業日になる場合あり（GMOあおぞら側の遅延ではない）。

→ 用途的には「振込から数秒〜十数秒で自社サーバーにPOST通知」が現実解。ポーリング不要。

## 3. 申請方法と所要期間
申込フロー（公式）:
1. 銀行API接続申込フォーム入力（オンライン）
2. 事前ヒアリングシート提出 + 接続審査
3. API利用契約書・申込書締結
4. 開発環境（サンドボックス相当）情報送付 → 接続試験
5. 本番接続開始

期間目安:
- 契約完了：**最短約1週間**
- 本番API接続完了まで：**約2ヶ月**
- サンドボックス sunabar：**即日・審査なし**で利用可（実装検証を先行できる）

必要書類: API利用契約書、申込書、ヒアリングシート（具体項目は申込後に送付）。

## 4. 技術仕様の要点
- **認証**: OAuth 2.0 + OpenID Connect（接続企業のシステム仕様に応じて選択可）
- **エンドポイント**: API開発者ポータル（`https://api.gmo-aozora.com/ganb/developer/api-docs/`）に仕様書公開
- **Webhook仕様書**: イベント通知編 v1.16.0（2024-07-15更新、BaaS版）
- **SDK**: GitHub `gmoaozora/gmo-aozora-api-php` で公式PHPサンプルあり（他言語は仕様書ベースで自前実装）
- **サンドボックス**: sunabar（実口座を使わずAPI挙動を検証可能）
- **レート制限**: 公式サイトに明示記載なし。仕様書（要登録）で確認が必要。
- **セキュリティ**: HMAC-SHA256 署名検証、アクセストークン検証

## 5. 代替案（参考）

| 銀行 | 法人API無料度 | 即時入金通知 | 備考 |
|---|---|---|---|
| **GMOあおぞらネット銀行** | ◎（28種無償） | ◎ Webhook | 国内銀行で公開API最多、デファクト |
| 住信SBIネット銀行（NEOBANK BaaS） | △（パートナー契約ベース、企業向け要相談） | ○ | API公開はあるが個別契約色が強い、無料枠は限定的 |
| みずほ銀行 / 三菱UFJ / 三井住友 | ×〜△（基本有料・初期費用高い） | ○（一部Webhook） | 接続料・月額が発生、中小事業者向けではない |
| PayPay銀行（旧ジャパンネット銀行） | △（API提供あり、入出金APIは有償傾向） | ○ | 個別見積、Optiens規模では割高 |

→ Optiens（個人事業主スケールの法人）にとっては **GMOあおぞら一択**。既に法人口座を保有している優位性も大きい。

## 推奨アクション（次の一手）
1. **即実行（今週）**: sunabar に代表アカウントで登録し、Webhook受信用エンドポイント（Vercel Functions or Supabase Edge Function）で署名検証PoCを完了。
2. **Phase A2着手と同時（〜2026年Q3）**: 本番API接続申込フォームを提出。並行してSupabaseに `bank_events` テーブルを設計し、freee MCPと連携。
3. **構成案**: バーチャル口座（顧客別発番）→ 入金 → Webhook → Vercel Function → Supabase記録 → freeeに自動仕訳 → Google Tasksに「入金確認済み」自動投入。
4. **コスト**: バーチャル口座無料・API無料・Vercel/Supabase既存枠内 → **追加月額 0円**で実装可能。
