# AI運用定着パッケージ 商品設計案 v0.1

## 1. 結論

Optiensの追加商品として、**AI運用定着パッケージ**を設計する。

ただし、いきなり高額な講座・コミュニティ商品にはしない。現行ファネルである「無料診断 → 詳細版AI活用診断 → AI診断官β / 単発AI相談 / 有償要件定義 → 導入支援 → 保守」に対し、以下の3商品を追加候補とする。

| 商品 | 位置付け | 価格案 | 主対象 |
|---|---|---:|---|
| AI運用定着スターター | 導入前・導入直後の4週間支援 | ¥120,000税抜 / ¥132,000税込 | まず社内でAI活用を始めたい小規模事業者 |
| AI運用定着スタンダード | 3ヶ月の実務定着支援 | ¥300,000税抜 / ¥330,000税込 | 導入支援後に社内運用を根付かせたい顧客 |
| 社内AI担当者育成ミニ講座 | 研修単体商品 | ¥66,000税込 | まだ導入支援までは不要だが、担当者を1人育てたい顧客 |

当面の推奨販売順は、**スターターのみ小さく開始 → 導入支援顧客向けにスタンダードを提案 → 需要が見えたらミニ講座を公開**。

---

## 2. 商品の役割

### 2.1 既存商品との違い

| 区分 | 目的 | 成果物 | 顧客の変化 |
|---|---|---|---|
| 詳細版AI活用診断 | 何をAI化すべきか判断する | Google Slidesレポート | 優先順位と費用感が分かる |
| AI診断官β | レポートをもとに次工程を整理する | 構造化メモ | 相談内容が要件に近づく |
| 有償要件定義 | 導入支援の範囲を固める | スコープ整理・概算 | 正式見積に進める |
| 導入支援 | 仕組みを構築する | システム・運用マニュアル | 業務が自動化される |
| 保守 | 構築済みシステムを改善・安定運用する | 修正・改善・報告 | 使い続けられる |
| AI運用定着パッケージ | 人・ルール・習慣を整える | ルール、テンプレ、練習メニュー、定着記録 | 社内でAIを使い続けられる |

AI運用定着パッケージは、システム構築そのものではなく、**AIを日常業務へ組み込むための人間側の運用設計**を扱う。

### 2.2 顧客に約束する状態

顧客に対して約束するのは、「AIで何でもできる」ではなく、以下の状態。

- どの業務でAIを使うか、使わないかが決まっている
- AIに渡してよい情報・渡してはいけない情報が整理されている
- よく使う業務テンプレートが最低5本ある
- 出力を誰が確認し、どこまで人間が承認するかが決まっている
- 月次で効果と課題を振り返れる
- 代表・店長・事務担当者の誰か1人が、AI活用の一次窓口になれる

## 3. 商品1: AI運用定着スターター

### 3.1 概要

| 項目 | 内容 |
|---|---|
| 価格 | ¥120,000税抜 / ¥132,000税込 |
| 期間 | 4週間 |
| 提供形式 | オンライン |
| 想定顧客 | 詳細版AI活用診断後、すぐ大きな導入支援に進むほどではない顧客 |
| 目的 | まず1〜3業務でAI活用を始め、社内に小さな成功体験を作る |
| 内部想定工数 | 6〜8時間 |

### 3.2 含むもの

| 内容 | 回数・範囲 |
|---|---|
| 初回キックオフ | 90分×1回 |
| 実務テンプレート作成 | 5本まで |
| AI利用ルール簡易版 | 1枚 |
| 業務別AI活用マップ | 1枚 |
| 週次フォロー | 30分×3回 |
| 非同期質問 | 期間中5件まで |
| 最終レポート | 1枚。継続判断・次の改善候補・導入支援化の要否 |

### 3.3 含まないもの

- 外部API連携
- 認証、個人情報、決済、本番システム構築
- 社内全体向け研修
- 既存システムの改修
- 24時間質問対応
- AIツール利用料、クラウド利用料

含まない範囲に入る場合は、スポット相談チケット、導入支援、保守へ切り替える。

### 3.4 4週間の進行

| 週 | ゴール | 作業 |
|---|---|---|
| Week 0 | 対象業務を決める | 申込フォーム、既存資料確認、事前ヒアリング |
| Week 1 | AI利用ルールを作る | キックオフ、使う業務/使わない業務、入力禁止情報、承認ルールを決定 |
| Week 2 | テンプレートを使い始める | 業務テンプレート5本を作成し、実務で試す |
| Week 3 | つまずきを潰す | 出力品質、社内の抵抗、入力例、確認フローを修正 |
| Week 4 | 継続判断する | 効果・課題・次月以降の選択肢を整理 |

### 3.5 標準テンプレート候補

業種に合わせて5本を選ぶ。

- 問い合わせ返信下書き
- 口コミ返信下書き
- SNS投稿案
- 議事録からTODO抽出
- 顧客への案内文
- 見積依頼内容の整理
- FAQ下書き
- 社内マニュアルの要約
- 仕入れ・発注メモの整理
- クレーム一次対応文の下書き

## 4. 商品2: AI運用定着スタンダード

### 4.1 概要

| 項目 | 内容 |
|---|---|
| 価格 | ¥300,000税抜 / ¥330,000税込 |
| 期間 | 3ヶ月 |
| 提供形式 | オンライン |
| 想定顧客 | 導入支援後、または複数業務でAI活用を根付かせたい顧客 |
| 目的 | 社内のAI利用ルール、テンプレート、振り返り、改善サイクルを固定化する |
| 内部想定工数 | 14〜18時間 |

### 4.2 含むもの

| 内容 | 回数・範囲 |
|---|---|
| 初回キックオフ | 90分×1回 |
| 月次定着MTG | 60分×3回 |
| 隔週ミニレビュー | 30分×3回 |
| 業務テンプレート作成 | 10本まで |
| AI利用ルール | 簡易規程1本 |
| AI活用ログシート | Google Sheets想定。利用件数、削減時間、差し戻し理由を記録 |
| 定着KPI設計 | 利用件数、削減時間、修正率、手戻り件数 |
| 非同期質問 | 期間中15件まで |
| 最終レポート | 3ヶ月後の成果、残課題、保守/導入支援/自走の判断 |

### 4.3 3ヶ月の進行

| 期間 | ゴール | 作業 |
|---|---|---|
| 1ヶ月目 | 使う業務を絞る | 業務棚卸し、AI利用ルール、テンプレート5本、活用ログ開始 |
| 2ヶ月目 | 現場の使い方を整える | テンプレート追加、出力確認ルール、差し戻し理由の整理 |
| 3ヶ月目 | 改善サイクルを固定する | KPI振り返り、次の自動化候補、保守/導入支援/自走判断 |

### 4.4 スタンダードで扱うKPI

| KPI | 定義 | 目安 |
|---|---|---|
| AI利用件数 | 週あたりAIを業務に使った回数 | 週5件以上 |
| 削減時間 | AI利用で短縮できた推定時間 | 月5時間以上 |
| 修正率 | AI出力を大幅修正した割合 | 30%以下を目指す |
| 差し戻し理由 | 使えなかった原因の分類 | 入力不足、ルール不足、業務不適合など |
| 継続候補業務 | 翌月も使う業務数 | 3業務以上 |

KPIは成果保証ではなく、運用判断のための観測値として扱う。

## 6. 販売導線

### 6.1 詳細版AI活用診断後

詳細レポートで「今すぐ大きな導入支援までは不要」と判断された顧客に、スターターを案内する。

案内文例:

> すぐにシステム構築へ進む前に、まず4週間でAIを使う業務・使わない業務・確認ルールを整えるプランです。テンプレート5本を一緒に作り、実務で試しながら定着の可否を判断します。

### 6.2 AI診断官β後

レビューで業務整理はできたが、導入支援ほどの予算やスコープがない場合に案内する。

判断基準:

- 業務自動化より、まず人の使い方が課題
- 外部API連携や本番運用は不要
- 顧客側にAI担当者候補がいる
- 月5〜10時間程度の削減余地はある

### 6.3 導入支援後

納品後に「運用マニュアルだけでは社内に浸透しない」顧客へ、スタンダードを提案する。

保守との使い分け:

- システム修正・機能追加が中心 → 保守
- 人の使い方・ルール・社内展開が中心 → AI運用定着スタンダード
- 両方ある → 保守 + スタンダード、または導入支援見積に内包

## 7. 受注可否の判断基準

### 7.1 受けてよい

- 代表・店長・事務担当者など、実際に使う人が1人以上参加できる
- 対象業務が1〜5個に絞れる
- 顧客が「AIだけで完全自動化」ではなく「人間が確認する運用」を受け入れている
- 個人情報や機密情報の取り扱いを慎重に進められる
- まず小さく試すことに納得している

### 7.2 受けない / 別商品へ回す

- 本番システム構築が主目的 → 導入支援
- 既存システムの修正が主目的 → 保守またはスポット相談チケット
- 24時間相談、質問無制限、社員全員研修を求める
- 成果保証、売上保証、採用保証を求める
- 投資、仮想通貨、副業案件獲得など、Optiensの主戦場から外れる

## 9. 公開コピー案

### 9.1 AI運用定着スターター

> AIを導入したいけれど、何から社内で使えばよいか分からない方向けの4週間プランです。使う業務・使わない業務・確認ルールを整理し、よく使うテンプレートを一緒に作ります。

### 9.2 AI運用定着スタンダード

> AI活用を一度きりの研修で終わらせず、3ヶ月かけて社内の実務に根付かせる支援です。利用ルール、業務テンプレート、活用ログ、月次振り返りを整え、継続判断まで伴走します。

### 9.3 社内AI担当者育成ミニ講座

> 社内でAI活用の一次窓口になる方を、短期間で育てるミニ講座です。AIへの依頼文、情報の扱い、出力確認、社内ルールの基礎を実務向けに整理します。

## 10. 初期リリース方針

### Phase 1: 非公開メニューとして検証

- 対象: 詳細レポート後、またはAI診断官β後の顧客
- 提供商品: AI運用定着スターターのみ
- 上限: 月2社まで
- 記録: 工数、質問件数、テンプレ作成数、顧客満足、導入支援化率

### Phase 2: 導入支援後オプション化

- 対象: 導入支援完了顧客
- 提供商品: スタンダードを追加
- 目的: 保守だけでは拾いにくい「人・ルール・習慣」の課題を受ける

### Phase 3: 公開ページ化を検討

以下の条件を満たしたら、公開ページまたはサービスページ内セクション化を検討する。

- スターター3件以上提供
- 内部工数が1件8時間以内に収まる
- 顧客側でテンプレートが継続利用される
- 導入支援または保守への転換が1件以上ある

## 11. リスクと対策

| リスク | 対策 |
|---|---|
| 研修だけで終わり、導入支援につながらない | 最終レポートに導入支援化の要否を必ず書く |
| 質問が無制限に膨らむ | 非同期質問件数を明記する |
| 成果保証と誤解される | KPIは観測値であり保証ではないと明記する |
| 保守との違いが曖昧になる | 保守はシステム、定着は人・ルール・習慣と整理する |
| 顧客の情報管理が危うい | 入力禁止情報と承認ルールを最初に決める |
| 代表1名体制で提供過多になる | Phase 1は月2社上限にする |
